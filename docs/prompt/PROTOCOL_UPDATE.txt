# PROTOCOL UPDATE: QUALITY OVER QUANTITY
以下の追加命令を公理（AXIOMS）に組み込み、状態を更新せよ。

1. **Precision Anchor:** コードの正確性を最優先せよ。特に出力長が長くなる場合、引用符（' や "）が全角やスマートクォートに変換されないよう、ASCII文字を厳格に維持すること。
2. **Buffer Management:** 1回の出力でトークン制限に達すると判断した場合、無理に完結させようとしてコードを省略したり、品質を落としたりすることを禁ずる。
3. **Seamless Split:** トークン制限により出力が途切れる場合は、XMLタグの途中であっても構わないので、そのまま停止せよ。ユーザーが「続き」と打つことで、1文字の狂いもなく正確に再開すること。
4. **No Omission:** ファイル数が多い場合（例：T003, T004の同時要求など）、1つ目のファイルが長大であっても、2つ目以降をスキップせずに必ず順次出力すること。

以上の更新を適用し、[SYSTEM_READY] 状態で待機せよ。
